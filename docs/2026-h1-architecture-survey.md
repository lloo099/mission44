# 2026 上半年前沿开源大模型架构:一份技术综述

> 这是一份对 2026 年 H1 中国前沿开源大模型(DeepSeek-V4、GLM-5、MiMo-V2.5-Pro、MiniMax M3、Kimi-K2.6、LongCat、Qwen3.6/3.7)的架构技术综述。重点不在"谁的跑分高",而在**这些模型在注意力、MoE、优化器、后训练、训推一致性上各自做了什么、为什么、代价是什么**。
>
> **关于信源**:文中区分三档可信度——**确证**(读过技术报告/官方 config)、**二手**(多源一致但未读一手)、**自报/待报告**(厂商自报或报告尚未发布)。凡涉及具体跑分,默认多为厂商自报、缺统一第三方复测,阅读时请打折。MiniMax M3 截至成稿技术报告尚未发布,其结构/训练细节为二手与图反推。

---

## 0. 一句话全局图景

2026 上半年这批旗舰,真正的竞赛不在"堆参数",而在**如何在规模与长上下文下省下算力、显存、带宽**。这条主线贯穿四个层面:**注意力**(怎么不为长上下文付二次方代价)、**MoE**(怎么把算力花在刀刃上)、**优化器**(怎么每步更高效)、**后训练 + 训推一致性**(怎么把能力稳定地灌进模型而不在 RL 时崩掉)。下面逐层拆。

一个会反复出现的诚实判断先放这:**架构创新大多是"效率/规模使能",不是"能力魔法"**——等算力下通常只有 5–15% 的差距,绝对能力更多由规模、数据、后训练决定。Qwen3.6 的 27B 稠密模型在编码上打败自家 397B MoE,就是"架构+训练 > 堆参数"的最好注脚。

---

## 1. 注意力的三大家族

标准注意力有两个长上下文痛点:计算上 O(n²)、显存上 KV cache 随长度线性增长。2026 这批模型对此的回答,可归为**三个根本不同的家族**,外加一条与注意力正交的 MoE 创新轴。

| 家族 | 机制 | 复杂度 | 代表 |
|---|---|---|---|
| ① 全注意力(不加稀疏) | MLA 或 GQA,attend 全部 | O(n²) | Kimi-K2.6(MLA) |
| ② 稀疏选择 | 只选一部分 token/块再算 | 次二次 | DeepSeek-V4(CSA/HCA)、GLM-5(DSA)、MiniMax M3(MSA)、MiMo(SWA) |
| ③ 线性注意力混合 | 线性注意力层 + 少量全注意力层 | O(n) | Qwen3.6(Gated DeltaNet) |
| (正交)MoE 算力分配 | 与注意力无关,改 MoE 怎么花算力 | — | LongCat-Flash(零计算专家) |

家族②内部还能再沿两个正交轴细分:**KV 底座(GQA vs MLA)× 稀疏方式(压缩/不压缩、选 token/选块)**。理解了这两个轴,下面所有具体机制就都能归位。

---

## 2. 底座之争:GQA vs MLA

这是注意力设计的"地基"问题,和上层稀疏方式正交。两者都为压缩 KV cache,但思路根本不同。

**GQA(分组查询注意力)——少存、共享真实 KV 头。** H 个 query 头分成 G 组,每组共享 1 个 KV 头(如 64 个 Q 头但只有 8 个 KV 头)。缓存的仍是**真实、完整维度的 K/V**,只是数量从 H 降到 G。它是 MHA→MQA(太激进)→GQA(折中)这条线的折中点。

**MLA(多头潜在注意力)——压成 latent、用时还原。** 不减头数,而是把每个 token 的 K、V **压成一个低秩 latent 向量**(如 512 维)缓存,算注意力时再上投影还原成各头的 K/V。缓存的是**压缩表示**,维度可比 GQA 还小、且与头数无关。代价是要处理**解耦 RoPE**(位置编码与上投影吸收不兼容,故拆成"无位置压缩部分"+"带位置 rope 部分",这就是 config 里 qk_nope+qk_rope 的由来)。

**效果对比(确证 + 解读):** 等 KV 预算下 **MLA 表达力严格更强**(MLA 能表达 GQA、反之不行:GQA 是"复制 K/V",MLA 是"每头不同的上投影")。但"理论更强"不自动兑现——GLM-5 报告里,裸 Muon 下 MLA 追不平 GQA-8,他们做了 **Muon Split** 才追平;同时 MLA 的 cache(576 维)比 GQA-8(2048 维)小约 3.5×。**真正的分水岭不是效果而是工程**:GQA 能直接复用 FlashAttention/vLLM/SGLang、实现简单、对 RL 训推一致更友好;MLA 压得更狠但要背 latent KV 的 kernel 工程债。所以 MiMo、MiniMax M3 选 GQA(production-ready 最低风险),DeepSeek 全系、GLM、Kimi、LongCat 选 MLA(极致压缩)。

一句话:**GQA 改"存几个头",MLA 改"怎么表示 KV";等预算下 MLA 略占理论上风但需配套,选择真正取决于你愿为压缩背多少工程债。**

## 3. 稀疏选择家族:同源于 NSA,四种分化

家族②全都能追溯到 DeepSeek 的 **NSA(Native Sparse Attention)** 的"选块"思想——只对相关的一部分上下文做注意力。NSA 本体有三条并行分支(压缩 + 选择 + 滑动窗口)加一个学习门控,后续各家把它沿不同方向简化:

| 方法 | KV 底座 | 选择粒度 | 是否压缩 KV | 注意力在哪算 | 代表 |
|---|---|---|---|---|---|
| **DSA** | MLA | token 级 | 是(latent) | 真实(MLA)KV | GLM-5、DeepSeek-V3.2 |
| **CSA/HCA** | MLA系/序列压缩 | 块级 | 是(4×/128×) | 压缩 KV | DeepSeek-V4 |
| **SWA 混合** | GQA | 固定窗口 | 否 | 真实 KV(局部) | MiMo |
| **MSA** | GQA | 块级 | 否 | 真实 KV | MiniMax M3 |

几个要点:

- **DSA(DeepSeek Sparse Attention)**:在 MLA 上加一个轻量 lightning indexer,为每个 query 选 token 级 top-k(如 top-2048)。GLM-5 通过"从稠密 MLA 基座继续预训练"加上 DSA,称其"构造上无损"。
- **CSA/HCA(DeepSeek-V4)**:更激进——CSA 4× 压缩 + 块选择,HCA 128× 重压缩 + dense,层间交替,且每层内嵌一个未压缩的 SWA 局部分支。把 KV 压到 V3.2 的约 10%,撑起 1M 上下文。
- **MiMo 的 SWA 混合**:最简,纯 GQA + 60 层滑动窗口(窗口 128)+ 10 层全局(6:1)+ 可学习 attention sink。一个反直觉的消融结论:**窗口 W=128 反而比 W=512 好**,因为小窗口起正则作用、并强制"局部归 SWA、长程归全局层"的清晰分工。
- **MSA(MiniMax M3)**:= "DSA 的动态块选择 + SWA 的不压缩"。两阶段"先挑再算":索引分支(单头共享 index key + Block Max Pool → TopK 选块,近零成本)→ 稀疏分支(对选中块在**真实未压缩 KV** 上算真注意力)。同组 query 共享一份选择,FlashAttention 可复用。稀疏率约 6–7%,1M 下有效感受野约 60–70k。(注:MSA 机制来自架构图反推,待官方报告证实。)

**一个重要且未解的争论:稀疏注意力哪种长上下文最强,三家一手证据互相打架。** GLM-5(选 DSA)在检索/RULER 上证明 DSA 无损、SWA 有 gap;MiMo(选 SWA)在 GSM-Infinite 带噪长推理上证明 hybrid SWA **胜过** DSA、称 DSA"在带噪长推理上有内在劣势";M3 的 MSA 想两头取优。**每家的消融都偏袒自家选择。** 合理结论:DSA 强在精确检索、hybrid 强在抗噪推理、MSA 想兼顾——**没有普适最优,取决于任务和长上下文模式**。

## 4. 第三条路:线性注意力(Qwen3.6)

Qwen3.6 走了和稀疏选择完全不同的路:**线性注意力混合**。它用 **Gated DeltaNet**(一种 O(n) 线性注意力,delta-rule 状态更新 + 门控,类似一个高效的循环记忆系统)与标准 **Gated Attention** 按 **3:1** 交错——27B 是 64 层 = 16 块 ×(3 个 DeltaNet 子层 + 1 个全注意力子层)。三分之三的层用线性注意力扛长程效率,每四层一个全注意力"锚定"精确的 token 级推理。线性注意力的 KV 不随长度增长,这是它和稀疏选择世界观上的根本区别。

值得对照的是 **MiniMax 走了反方向**:M1 用过线性(Lightning Attention)、M2 整代回到全注意力(官方甚至专门发博客《Why Did M2 End Up as a Full Attention Model?》)、M3 又回归稀疏。**Qwen 押注线性、MiniMax 放弃线性**——同一时间两条路的分歧,说明这个方向远未盖棺。

另外 Qwen3.6-27B 是**稠密**模型,却在 SWE-bench Verified 上(77.2)打败自家 397B MoE(76.2),并在 Terminal-Bench 2.0 上追平 Claude Opus 4.5。这是"架构+训练 > 堆参数"最干净的一个实证。

## 5. MoE 的另一条创新轴:LongCat 的零计算专家

不是所有创新都在注意力上。**LongCat-Flash** 的招数全在 MoE:**零计算专家**——512 个真专家 + 256 个"零专家"(恒等/空操作),top-12 路由,用一个 PID 控制器把平均激活维持在约 27B(560B 总参里)。效果是**按 token 重要性动态分配算力**:简单 token 路由到更多零专家、少算,难 token 多算。这是这批模型里独一份的"动态算力"。配套的 **ScMoE(Shortcut-connected MoE)** 把专家通信藏进计算里重叠,>100 TPS@H800。

它的注意力反而是标准 MLA——说明**创新可以只在 MoE 这条轴上发生**,与注意力家族正交。

## 6. 优化器:Muon 家族

前沿 MoE 预训练的优化器正在从 AdamW 向 **Muon 系**迁移(DeepSeek-V4 用 Muon、GLM-5 用 Muon Split、Kimi-K2.6 用 MuonClip;但 MiMo 仍用 AdamW,未普适)。Muon 值得单独讲透,因为它的原理常被误解。

**机制(三步):** 对一个 2D 权重矩阵 W,① 攒动量 M ← μM + G;② **正交化** M——做 SVD M=UΣVᵀ,把奇异值全拍成 1,用 **O=UVᵀ** 替换 M(即取离 M 最近的半正交矩阵);③ 更新 W ← W − ηO。每步真做 SVD 太贵,故用 **Newton-Schulz 迭代**(只含矩阵乘的多项式迭代,约 5 次)近似 UVᵀ,可跑 bf16。

**澄清两个常见误解:**
- Muon 和 Adam **都是一阶方法,都不求二阶梯度(Hessian)**。Muon 砍掉的是 Adam 的**二阶矩 v**(=梯度平方的滑动平均,一个一阶统计量,与二阶导无关),所以优化器状态只一份、省一半显存。
- **求梯度这一步两者完全相同**(同一次反向、同一个 G)。区别只在"拿到 G 之后":Adam 逐元素做 m/(√v+ε),Muon 把整个矩阵正交化。唯一对梯度"看法"不同的是——Adam 把梯度当一堆独立标量,Muon 把它当一个有奇异值结构的矩阵。

**为什么正交化有用:**
1. **它是"谱范数下的最速下降"。** 普通 GD 是欧氏范数下的最速下降,但权重矩阵对网络的影响由**谱范数**(放大能力)决定;在谱范数约束下求最速下降,最优方向恰好就是 −UVᵀ。即"用对这一层真正有意义的范数来量步长"。
2. **把更新在所有奇异方向上调匀(预条件)。** 原始梯度高度各向异性,正交化让每个奇异方向拿等量步长(更新条件数=1),每步用满矩阵全秩,信息利用率最高。

**为什么不是所有参数都用 Muon:** 因为 Muon 干的事是"扔掉幅度、只留方向",这只在"幅度是噪声"处才对。
- 1D 参数(bias、norm 缩放、标量):没有矩阵可正交化,无定义,交给 Adam。
- Embedding / LM head:技术上是 2D,但每行对应一个 token,梯度稀疏且词频呈 Zipf 分布,**行间幅度差异是真信号**(高频/低频 token 本该被不同更新),正交化会把它抹平;且词表维度巨大、在 Muon 不帮忙处跑最贵的正交化,纯亏。实测 Adam 更好。
- 所以**标准配方本就是混合**:隐层 2D 权重走 Muon,embedding/head/norm/标量走 Adam。"全用 Muon"从来不是它的主张。

**效果(确证 + 校准):** 起源就靠刷训练速度纪录出名;Moonshot 把它扩到大 MoE,报告约 **2× token 效率**;最强证据是三家前沿实验室独立采用它做旗舰预训练。但限定三条:① "2×"setup 相关、非普适常数;② 大规模需配稳定化(裸 Muon 会注意力 logit 爆炸,故有 **MuonClip = Muon + QK-Clip** 钳住 logit、**Muon Split** 按头正交化让 MLA 追平 GQA);③ **主要是预训练的优势,RL 后训练仍以 AdamW 为主**。

## 7. 后训练范式:OPD/MOPD 的收敛

一个值得注意的收敛现象:**DeepSeek-V4(OPD)和 MiMo(MOPD)独立地走到了同一配方**——先用 RL 训出若干领域专家,再用 **on-policy 蒸馏**把多个教师合并进一个学生(token 级 reverse-KL 密集信号 + 结果奖励)。GLM-5 的"顺序 RL + 跨阶段蒸馏"思路相通(都为防灾难性遗忘/see-saw)。两家独立收敛,强烈暗示这是正在成型的标准后训练范式。

**但要诚实看它的真实战果**(来自 MiMo-V2-Flash 报告的一手数据):on-policy 蒸馏在**可验证域**确实让学生匹配甚至略超最强教师(AIME +0.2、HMMT +1.8 超过 RL 教师),但在 **agentic 搜索和主观域明显掉队**(BrowseComp 比教师低 6.3、创意写作倒退 3.9)。论文宣称"全域保住教师峰值"被它自己的表打了脸。**结论:OPD/MOPD 擅长收拢可验证域,agentic/主观域仍有 gap。**

Agentic 能力的真瓶颈也值得点明:不是算法,而是**真实环境 RL 的基础设施**(沙箱、并发、partial rollout)——DeepSeek-V4 的 DSec 沙箱(10 万+并发)、MiMo 的万级 K8s 环境,才是 agentic RL 能 scale 的关键。

## 8. 被低估的工程主线:训推一致性

只要做 RL,**训练和推理用同一个策略算出的 log-prob 必须对得上**,否则训练信号失真、容易崩。这是整批模型共同处理、却很少被外部讨论的暗线,而且解法分在三个层次:

- **算子层(DeepSeek-V4)**:batch-invariant 确定性内核 + TileLang DSL,做到训推逐位一致。
- **量化层(GLM-5)**:INT4 QAT,让量化后训推逐位一致。
- **路由层(MiMo)**:**R3(Rollout Routing Replay)**——RL 训练时复用 rollout 阶段记录的专家路由,解决 MoE 因数值精度导致的路由不一致,再配请求级 prefix cache。

共同的风险点是**离散选择操作**(DSA/MSA 的 top-k、MoE 路由)——它们在训推两侧极易因数值差异分叉。这也是 GQA + FlashAttention 比 MLA 更受 production 团队欢迎的隐性原因之一:数值路径更标准、更易对齐。

## 9. RL 里那些可以融合的算子

RL 相比 SFT 多出一整条 logits→log-prob→ratio→loss 的长尾计算链,几乎全是可融合的逐元素/规约操作。按杠杆从高到低:

1. **LM head + log-prob 融合(最高杠杆)**:RL 需要逐 token log-prob,通常还要算 π_new/π_old/π_ref 三份。朴素实现要物化完整 logits([B,S,V],15 万词表下单份就 ~20GB),但你只要每个位置一个标量。把 lm_head matmul 沿 vocab 维**分块**、内联做 logsumexp 在线累积 + 目标 token gather,**从不物化完整 logits**(Liger Kernel 的 fused linear+cross-entropy 那族)。这通常就是 RL 训练显存峰值的决定项。
2. **同一遍扫描顺手算出熵和全分布 KL**:边际成本极低。OPD/MOPD 的 reverse-KL 蒸馏 loss 本质就是这条链,可做成一个 fused "linear + 蒸馏 loss" kernel。
3. **RL loss 的逐元素链一锅端**:ratio→clip→×advantage→min→mask→归约,加上 k1/k2/k3 KL 估计、训推重要性采样的 ratio 截断,全可融进一个 kernel。
4. **主干标配**:FlashAttention(融合的祖师爷)、RMSNorm+残差、SwiGLU、RoPE 内联、fused 优化器更新。
5. **rollout 侧**:采样链融合(temperature→top-k/p→softmax→sample)、MoE 的 router softmax+topk 融合 / permute-unpermute 融合 / grouped GEMM。

**一个必须挂的 caveat**:融合会改变浮点规约顺序——训练侧融了、推理引擎没融(或融法不同),log-prob 失配会变大。所以**融合和训推一致性是同一本账**:两侧数值路径要么都融、要么对齐,追求逐位一致还得让 kernel 做成 batch-invariant。

## 10. 几条诚实的总判断

把整批模型看完,有几条比单个 spec 更值得记住的判断:

1. **架构 ≠ 能力。** 架构创新多是效率/规模使能(等算力 ~5–15%),绝对能力更多由规模+数据+后训练决定。Qwen3.6-27B 稠密胜 397B MoE 是活例。
2. **长上下文注意力没有普适最优。** 全注意力 / 稀疏选择 / 线性,三家世界观之争;稀疏内部 DSA/SWA/MSA 也各有所长,取决于检索 vs 抗噪推理 vs 兼顾。别被任何单方消融带跑。
3. **厂商跑分要打折。** 这批模型的强势数字大量是自报、缺统一第三方复测(M3 尤甚,连技术报告都还没出)。
4. **信源纪律的一个教训:GLM"完全在昇腾训练、零 NVIDIA"是误传。** GLM-5 一手报告并未声明训练硬件、训练基建引用 Megatron-LM、未提 MindSpore;第 5 章讲的是把模型**适配推理**到 7 个国产平台,而 GLM-5.1 的加载明确需要 NVIDIA B200。诚实画像是"推理可上 B200 + 适配 7 国产平台,训练硬件未明示"。**国产芯片做推理是真的,做训练更难且少有公开实证。**
5. **前沿正在分化成两条路。** 这批的"最新旗舰"已不全是文本推理 MoE:一边是**闭源 agent 旗舰**(Qwen3.7-Max,API-only、架构不公开,与自研芯片真武 M890 协同),一边是**原生全模态**(LongCat-Next 的 DiNA 离散统一、Kimi-K2.6/M3 的多模态化)。纯文本推理架构的横向对比,只是 2026 上半年的一个切片。
